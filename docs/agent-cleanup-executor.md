# Managed Residue Cleanup

SKS exposes only the current command and dollar-route catalogs. A normal `sks doctor` reports whether SKS-owned retired residue exists; `sks doctor --fix` removes that managed residue and rewrites managed state so the retired surface cannot be listed, redirected, or resumed.

The same reconciliation runs as part of the update migration before the updated installation is accepted. Cleanup covers generated skills and role files, managed mission/runtime artifacts, stale managed state fields, generated bridge manifests, and obsolete managed report files.

Safety is ownership-aware:

- files proven to be generated by SKS are removed physically;
- user-authored collisions are moved to `.sneakoscope/quarantine/` instead of being deleted;
- unknown or unverifiable ownership fails closed;
- cleanup reports expose counts and blockers, not a customer-facing catalog of retired names;
- an update cannot report the current surface as reconciled while managed residue remains.

Operator flow:

```bash
sks doctor --json
sks doctor --fix --yes --json
sks update now --json
```

After reconciliation, `sks commands --json` and `sks dollar-commands --json` are the authoritative public catalogs. Unknown commands remain unknown and receive no compatibility redirect.
