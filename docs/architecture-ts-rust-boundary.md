# Architecture: TypeScript / Rust boundary

SKS has two execution surfaces. TypeScript is the runtime source-of-truth; Rust
is an optional, side-effect-free accelerator. This document defines the boundary
and the gates that enforce it.

## TypeScript is the source of truth

- All runtime logic lives in `src/**/*.ts` and compiles to `dist/**`.
- `dist/bin/sks.js` is the single CLI entry point.
- There are **no** `src/**/*.mjs` runtime mirrors. The only `.mjs` files are
  release gates and tests under `scripts/` and `test/`, which import from
  `dist/**`.
- `dist/**` is build output: generated by the TypeScript build and **never**
  hand-edited. Anything that needs to change is changed in `src/**/*.ts` and
  rebuilt.

## Rust is an optional accelerator

- The Rust crate lives in `crates/sks-core`, with an optional `sks-rs` binary.
- Rust handles **pure computation only**: hashing, JSON validation, secret
  scanning, voxel validation. It performs no I/O side effects.
- Rust **never** mutates files, rewrites config, or performs publish/build
  side-effects. Those operations are exclusively TypeScript's responsibility.
- Every Rust-accelerated path has a JS/TS fallback. If the binary is missing,
  the JS/TS implementation runs instead.
- Version-mismatched binaries are rejected and routed to `js_fallback`, so a
  stale or incompatible `sks-rs` can never produce wrong results.

## Toolchain independence

- `npm i -g sneakoscope` and `sks doctor` work with **no** Rust toolchain
  installed — the JS/TS fallbacks cover every path.
- `npm publish` / the build pipeline **never** compiles cargo. Rust is built and
  distributed separately as an optional optimization; it is never a prerequisite
  for installing or running SKS.

## Enforcing gates

| Gate | Enforces |
|------|----------|
| `architecture:guard` | every pipeline/gate carries the SSOT guard and default SOLID expectations: SRP, OCP, LSP, ISP, and DIP |
| `runtime:ts-source-of-truth` | `src/**/*.ts` is the authoritative runtime source |
| `runtime:no-src-mjs` | no `src/**/*.mjs` runtime mirrors exist |
| `runtime:dist-parity` | `dist/**` matches the compiled `src/**` (never hand-edited) |
| `runtime:ts-rust-boundary` | Rust does pure computation only — no file mutation, config rewrite, or publish side-effects |
| `rust:smoke` | optional Rust paths have working JS/TS fallbacks and reject version-mismatched binaries |
