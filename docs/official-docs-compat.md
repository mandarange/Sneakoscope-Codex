# Official Docs Compatibility

SKS 1.18.7 keeps release-gated behavior tied to current official source expectations for Codex hooks, Codex structured `--output-schema` usage, Codex Chrome Extension web verification, ChatGPT Images 2.0 / OpenAI Image Generation with `gpt-image-2`, OpenAI Structured Outputs, Codex 0.134 native agent/profile/MCP semantics, and inherited Codex 0.133 multi-session semantics.

The compatibility report is generated by:

```bash
npm run official-docs:compat
```

It writes:

- `.sneakoscope/reports/official-docs-compat-1.18.7.json`
- `.sneakoscope/reports/official-docs-compat-1.18.7.md`

The report records the current baselines used by release readiness:

- OpenAI Codex release baseline: `rust-v0.134.0`
- Inherited Codex baselines: `rust-v0.133.0`, `rust-v0.132.0`
- Codex App image generation feature guide: `https://developers.openai.com/codex/app/features#image-generation`
- Codex Chrome Extension guide: `https://developers.openai.com/codex/app/chrome-extension`
- ChatGPT Images 2.0 system card baseline: `https://deploymentsafety.openai.com/chatgpt-images-2-0`
- ChatGPT Images 2.0 announcement reference: `https://openai.com/index/introducing-chatgpt-images-2-0/`
- OpenAI Image Generation guide: `https://developers.openai.com/api/docs/guides/image-generation`
- `gpt-image-2` model guide: `https://developers.openai.com/api/docs/models/gpt-image-2`
- OpenAI Structured Outputs guide: `https://developers.openai.com/api/docs/guides/structured-outputs`

Release-gated rules:

- Codex hook parity uses the latest vendored 10-event hook schema and blocks warning-producing output shapes.
- Codex structured extraction prefers session-preserving `codex exec resume --output-schema` when available.
- Codex 0.134 profile selection, local history search, MCP environment/OAuth/schema/readOnlyHint behavior, subagent hook context, and managed proxy propagation are mapped in the release matrix.
- Codex 0.133 goal defaults, permission profiles/requirements, plugin discovery, marketplace config, and remote-control foreground app-server behavior remain inherited compatibility rows.
- Full SKS image evidence uses the official Codex App `$imagegen`/`gpt-image-2` surface; OpenAI API or codex-lb fallbacks are non-Codex evidence and must be explicitly labeled.
- Web, browser, localhost, website, webapp, and web-based app verification uses the official Codex Chrome Extension path first; Computer Use is reserved for native Mac/non-web targets.
- Newest-model image prompts explicitly request ChatGPT Images 2.0 / GPT Image 2.0 with `gpt-image-2`; public X/social reports can inform prompt heuristics but are not capability or evidence specs.
- `gpt-image-2` image requests omit `input_fidelity`; high-fidelity image input handling is automatic for this model.
- UX/PPT generated-review routes must produce request/response artifacts, generated image artifacts, and strict structured issue ledgers before verified claims can pass.
- Structured extraction schemas use strict JSON schema behavior with `additionalProperties:false`.

If official Codex hook hashes are unavailable from the installed Codex CLI, SKS marks official discovery as integration-optional and enforces the managed-hooks path instead of writing SKS-only `trusted_hash` values.
