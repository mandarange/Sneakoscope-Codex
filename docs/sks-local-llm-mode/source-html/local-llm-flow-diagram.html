SKS + Local LLM Mode

Keep cloud-grade judgment
while your Mac handles fast local drafts
1. SKS
Breaks work into
safe decision units
SKS organizes the request, policy, code context, and verification criteria first.
Role: orchestration, final judgment, and safety gates
2. Local LLM
Handles drafts
and repeated passes
The local model helps with code reading, candidate drafts, summaries, and small refactor suggestions.
32GB+ RAM: 35B/31B class; below 32GB: start with 12B class
3. Cloud + Review
Applies important
decisions after review
High-risk changes, current facts, and final quality decisions stay inside the SKS verification loop.
Outcome: local drafts are used inside the SKS verification flow
Operating principle: the local LLM is a helper engine beside SKS, not a replacement model.
Start example: run llama-server -hf <model>:<quant>, then connect SKS local mode.
