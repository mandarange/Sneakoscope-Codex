SKS Local LLM Mode

A workflow where SKS keeps cloud-grade judgment while a local model on your Mac assists with repeated reading, drafting, and summarization.

1SKS decomposes the work and keeps the safety criteria in place.
2The local LLM handles fast candidate drafts and repeated passes.
3Important decisions and final application still pass through verification.
Local LLM Mode Introduction01 / 08
Concept

The concept is not replacement. It is a helper engine beside SKS.

What the local LLM is good at

  • Scanning long files and drafting candidate code or copy
  • Repeated summaries, comparison tables, and checklist drafts
  • Local pre-exploration and work breakdown support

What SKS still owns

  • High-risk decisions around policy, security, and databases
  • Current documentation checks and source-backed verification
  • Final integration, tests, and Honest Mode reporting

The clearest framing is “fast local drafts plus SKS verification.” Also tell users that final judgment and verification may still process required context through the existing SKS flow.

How it works

The workflow is simple

Step 1

Start a local server

Prepare an OpenAI-compatible endpoint with llama-server, Ollama, LM Studio, or a similar tool.

Step 2

Connect SKS

Set the local endpoint and model name so SKS can call the helper model.

Step 3

Review before applying

Local drafts are not trusted blindly; they are integrated after tests and review.

Setup

Recommended start: run a local server, then enable SKS local mode

1. Prepare the local model server

brew install llama.cpp

The Hugging Face model cards also document this macOS-friendly flow.

2. Connect SKS local mode

sks with-local-llm on \ --provider openai-compatible \ --model <model> \ --base-url http://127.0.0.1:8080

OpenAI-compatible means SKS calls the local server through an OpenAI-style API.

llama-server example

llama-server -hf <repo>:<quant>

This usually pairs with the openai-compatible provider and http://127.0.0.1:8080.

Check status and turn it off

sks with-local-llm status --json sks with-local-llm off

The setting is stored in ~/.sneakoscope/local-model.json.

The first run can take time because the model may need to download and populate local caches.

Announcement copy

Suggested user-facing announcement

Draft announcement copy

SKS now includes a mode for working alongside a local LLM. It keeps SKS verification and orchestration in place while letting a model running on your Mac assist with repeated code reading, summarization, and drafting. For Macs with 32GB+ RAM, start with Qwen3.6 35B or Gemma 4 31B GGUF. Below 32GB, start with Gemma 4 12B GGUF.

Core benefit

Faster repeated drafting and inspection loops

Operating rule

Local output is applied only after verification

Best fit

Developers who use SKS frequently on a Mac

Caveats

  • Local model performance depends on the Mac chip, memory, context length, and quantization.
  • Sensitive changes, database work, security, and current facts should stay in the SKS verification flow.
  • Before starting, run sks with-local-llm status --json to check provider, model, and base URL.

Before final distribution, check the SKS local-mode command names and setting keys against the current release docs.
