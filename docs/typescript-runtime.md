# TypeScript Runtime Policy

SKS 1.17.0 treats `src/**/*.ts` as the only runtime source of truth.

The npm runtime is generated by `npm run build` and executes only `dist/**/*.js`, with `dist/bin/sks.js` as the package entrypoint. Source imports in TypeScript keep NodeNext `.js` specifiers so the compiled `dist` tree can run without a source shadow copy.

Release checks enforce this contract with `runtime:no-src-mjs`, `runtime:ts-source-of-truth`, and `runtime:dist-parity`. The dist build manifest records the package version, source digest, source file count, source list hash, compiled JS count, compiled declaration count, and `src_mjs_runtime_files=0`.
