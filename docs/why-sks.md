# Why SKS

SKS is useful when the task needs a clear answer to one question: can this Codex route honestly be called complete?

It answers with artifacts rather than vibes:

- route contracts
- evidence indexes
- Completion Proof
- image voxel anchors for visual claims
- DB/hook/secret safety gates
- hermetic and black-box package checks
- rollback and managed-path visibility

The design goal is not the broadest harness surface. It is the shortest auditable path from user intent to verified or honestly blocked completion.
